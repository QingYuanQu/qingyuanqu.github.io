内部表

  • 源文件和目标文件夹在同一个文件系统中时，加载数据后，原文件会移动到目标文件夹中。

  • 如果是采用了LOCAL关键字，源文件只会拷贝本地Linux文件系统中的文件到目标文件夹中。

外部表

指向在 HDFS 中已存在的数据；

它和内部表在元数据的组织上是相同的，而实际数据的存储则有较大的差异；

外部表只有一个过程，加载数据和创建表同时完成，并不会移动到数据仓库目录中，只是与外部数据建立一个链接。当删除一个外部表时，仅删除该链接；

HIVE表与实际数据的对应关系，在 MySQL - hive DB - TBLS, SDS 表中。

需要注意：location需要是目录。

分区表

在 Hive 中，表中的一个 Partition 对应于表下的一个目录，所有的 Partition 的数据都存储在对应的目录中

桶表

桶表是对数据进行哈希取值，然后放到不同文件中存储。

insert 语句会触发MapReduce，可在YARN中观察到YARN Application状态。

相关日志

分区表与分桶表的区别

  • 分区是指基于表的某些列将数据分割成不同的存储单元；如根据日期分成多个分区目录(year=xxxx,month=xx,day=xx)
  • 而分桶是指将数据根据哈希函数分成一组固定的桶。
  • 分区不依赖MapReduce的Partitioner， 分桶依赖Partitioner。

视图

  • 视图是一种虚表，是一个逻辑概念；可以跨越多张表

  • 视图建立在已有表的基础上, 视图赖以建立的这些表称为基表

  • 视图可以简化复杂的查询，但不能提高查询的效率

1.为hive提供了一种允许客户端远程访问的服务。

2.基于thrift协议，支持跨平台，跨编程语言对hive访问；
